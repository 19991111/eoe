## EOE 项目 (演化智能体) - 核心原则

### 核心原则：只设计环境压力，不设计大脑结构
- 智能必须从持续的生存竞争中涌现，而非人为预设
- 预设节点类型(SENSOR, DELAY等)是"先天结构"，加速自然选择
- 精英选择机制本身也是一种环境压力

### 2026-03-09 v0.74 纯环境设计

**纯生存适应度模式** (`pure_survival_mode=True`):
```python
fitness = 食物数 × 1000  # 人工设计权重已移除
```

**元节点压缩** (终身学习):
- 追踪节点共同激活次数 → coactivation
- 当两节点共同激活≥10次 → 压缩为META_NODE
- META_NODE输出 = 父节点激活的加权组合
- 这是真正的"涌现新组件"！

### 2026-03-10 v0.97 三大突破机制

**专家建议的三个方案**:
1. 代谢疲劳 + 安全掩体 (降维6步长链路)
2. 无聊信息素 (破解气味盲区)
3. 夏日食物热力学 (当下收益)

**关键教训: 适应度悬崖**

最初实现"强制睡眠+20%死亡率"，Agent在完全无法控制的情况下随机死亡，演化无法学习。**这就是典型的适应度悬崖问题**。

**解决方案: 渐进式物理压迫**
- 疲劳度只影响速度，不强制睡眠
- 静止3帧后自动恢复疲劳
- 睡觉消耗大量能量（起床饥饿）
- 睡眠时食物自动掉落（物理贮粮）

**核心改造**:
- `environment.py`: 添加疲劳系统、信息素系统、食物热力学
- `agent.py`: 添加fatigue, max_fatigue, stationary_frames
- Flag控制: enable_fatigue_system(), enable_pheromone_system(), enable_food_thermodynamics()

**结果**:
- v0.97 最高适应度: 245.7
- 贮粮(Store): 出现

**原则**: 只设计环境压力，不设计大脑结构
