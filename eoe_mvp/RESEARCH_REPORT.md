# EOE项目研究报告：演化智能体的涌现与进化

## 一、项目目标

**终极目标**：通过纯演化（无人工设计）涌现出类人通用智能

核心问题：如何让Agent通过环境压力自然涌现出规划、推理等高级智能能力？

---

## 二、核心原则

### 2.1 核心原则（不可违背）

> **"只设计环境压力，不设计大脑结构"**

- 不预设任何神经网络结构
- 不预设任何功能模块
- 智能必须从"生存压力"中自然涌现
- 演化算法负责结构选择，环境负责功能筛选

### 2.2 红皇后假说

> "必须全力奔跑，才能保持在原地"

- 引入竞争性Agent（敌对）增加选择压力
- 敌对Agent协同进化（每N代刷新，使用当前最佳结构）
- 压力促进智能涌现

---

## 三、技术架构

### 3.1 基因组结构

```python
# 神经网络基因组
- 节点：传感器(SENSOR)、执行器(ACTUATOR)、隐层节点
- 边：权重连接（可启用/禁用）
- 可塑性：部分边有Hebbian学习能力（learning_rate > 0）
```

### 3.2 环境系统

| 机制 | 实现 |
|------|------|
| 季节系统 | 夏天食物充足，冬天无食物+高代谢 |
| 巢穴机制 | Agent可存储食物到巢穴 |
| 敌对Agent | 红皇后假说实现，竞争有限资源 |
| 课程学习 | 分阶段施压（轻松→正常→严峻） |

### 3.3 演化机制

- 精英选择：Top 20%存活并繁殖
- 结构突变：添加节点、添加边、调整权重
- 可塑性突变：启用边的Hebbian学习能力
- 冠军继承：每代保留最佳Agent作为参考

## 五、核心挑战与问题

### 5.1 贮粮行为的困境

**现象**：贮粮行为可以偶尔涌现（最多5个），但无法稳定遗传

**原因分析**：
```
贮粮需要6步同时突变：
  发现食物 → 拾取 → 携带 → 找巢穴 → 放下 → 冬天取回
任何一步失败，整个"贮粮基因"无法传承
```

这是超长链路行为的固有挑战

### 5.2 pickle保存问题

- 保存的Agent是"实验结束后的死亡状态"
- 运行时状态（能量、位置等）丢失
- 无法直接测试保存的冠军

### 5.3 规划能力缺失

- 当前模型有PREDICTOR节点（预测下一步传感器）
- 但缺乏"预测未来价值"的能力
- 奖励信号是即时的（反射），不是延迟的（规划）

### 5.4 环境复杂度不足

当前环境相对简单：
- 只有觅食和生存压力
- 缺乏需要多步规划才能解决的挑战

### 7.1 智能涌现的条件

## 更新：2026-03-10 晚间

### 重大突破：吃撑惩罚机制

**专家建议**：适应度悬崖 → 适应度缓坡

**核心机制**：
- 胃容量 = 1（吃1个就饱）
- 吃饱后额外能量消耗（惩罚）
- 携带食物无惩罚
- **涌现结果**：Agent学会"吃饱后不吃了 → 携带"

### 实验结果

| 版本 | 机制 | 适应度 | Carry | Store |
|------|------|--------|-------|-------|
| v0.79 | 课程学习 | 11998 | ❌ | ✅5 |
| v0.83 | 胃容量限制 | 1999 | ✅3 | ✅2 |
| v0.85 | +携带惩罚 | 2999 | ✅3 | ❌ |

### 里程碑

🎉 **携带行为 0→1 涌现！**

这是从"反射"向"规划"的关键一步。

### 核心洞察

通过生理学约束（胃容量）创造适应度缓坡：
- 吃饱后吃=惩罚（跑得慢/被杀掉）
- 吃饱后携带=无惩罚（可以跑）
- 这就是"规划"的雏形
