# EOE 版本管理

## 版本号规范

```
格式: MAJOR.MINOR.PATCH
- MAJOR: 架构重大变更 (如 v12 从零重构)
- MINOR: 新机制/新功能 (如 v11.1 压力熔炉)
- PATCH: Bug修复/参数调整

示例:
- v0.64 → v0.65 (新增机制)
- v10.4 → v11.0 (架构升级)
- v11.0 → v11.1 (新增功能)
```

## 版本升级流程

1. **PATCH (小修复)**: 直接修改,更新本文件
2. **MINOR (新功能)**: 
   - 更新版本号
   - 添加入版本历史
   - 如有需要创建新的运行脚本
3. **MAJOR (架构变更)**:
   - 创建新目录结构
   - 更新 MIGRATION_GUIDE
   - 归档旧版本
