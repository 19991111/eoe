# 记忆管理细则

> 项目记忆的组织、使用与维护规范

---

## 一、记忆体系

### 1.1 记忆类型

| 类型 | 文件 | 用途 | 生命周期 |
|------|------|------|----------|
| **长期记忆** | MEMORY.md | 核心原则、架构、版本 | 永久 |
| **每日记忆** | memory/YYYY-MM-DD.md | 日常记录 | 1年 |

### 1.2 记忆层级

```
MEMORY.md (顶层)
    ↓
memory/YYYY-MM-DD.md (日常)
    ↓
实时对话 (当前会话)
```

---

## 二、MEMORY.md 结构

### 必需章节

1. **项目概述** - 使命、哲学、研究方向
2. **版本体系** - 当前版本、历史
3. **架构体系** - 核心模块、物理法则
4. **管理细则** - 文档/脚本/大脑/记忆
5. **目录结构** - 完整树形结构
6. **硬件环境** - 关键路径
7. **待办事项** - 未来计划
8. **重要原则** - 核心规范

---

## 三、每日记忆格式

### 3.1 文件命名
```
memory/YYYY-MM-DD.md
例如: memory/2026-03-12.md
```

### 3.2 每日模板

```markdown
# YYYY-MM-DD 工作日志

## 今日完成
- [x] 任务1
- [x] 任务2

## 遇到的问题
- 问题描述

## 解决方案
- 解决方式

## 明日待办
- [ ] 待办1
- [ ] 待办2
```

---

## 四、写入时机

### 必须写入

| 事件 | 目标文件 | 内容 |
|------|----------|------|
| 版本发布 | MEMORY.md | 更新版本历史 |
| 架构变更 | MEMORY.md | 更新架构体系 |
| 新增机制 | MEMORY.md | 物理法则表 |
| 重大决策 | MEMORY.md | 重要原则 |
| 每日工作 | memory/YYYY-MM-DD.md | 完整日志 |
| 实验结果 | memory/YYYY-MM-DD.md | 数据记录 |
| 代码清理 | memory/YYYY-MM-DD.md | 变更记录 |

### 推荐写入

- 小bug修复 (累计后写入)
- 常规实验 (结果显著时写入)
- 学习新知识 (相关时写入)

---

## 五、内容规范

### 5.1 语言
- 文档: 中文为主,英文术语保留
- 代码: 英文
- 注释: 中文

### 5.2 格式
- 标题: `#` 一级, `##` 二级
- 列表: `-` 或 `1.`
- 代码: ` ```python ` 块
- 引用: `> ` 块

### 5.3 链接
- 内部: 相对路径
- 外部: 完整URL

---

## 六、维护规则

### 6.1 定期整理

| 周期 | 操作 |
|------|------|
| 每日 | 更新当日记忆 |
| 每周 | 合并到MEMORY.md |
| 每月 | 清理旧记忆 |

### 6.2 清理标准

- 超过1年的每日记忆可移至 `_archive/memory/`
- 过时的待办事项应删除或完成
- 重复内容应合并

### 6.3 质量检查

- [ ] 章节完整
- [ ] 链接有效
- [ ] 格式统一
- [ ] 无错别字

---

## 七、备份与恢复

### 7.1 备份
记忆文件已包含在项目仓库中,无需额外备份。

### 7.2 恢复
从git历史恢复:
```bash
git checkout HEAD -- memory/YYYY-MM-DD.md
```

---

## 八、特殊情况

### 8.1 会话中断
如果工作未完成,使用以下格式标记:
```markdown
## 待续...
- [ ] 未完成的任务
- [中断原因]
```

### 8.2 重要发现
使用强调标记:
```markdown
### 🎯 重要发现
> 关键内容
```
