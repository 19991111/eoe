## EOE项目 - 关键Bug修复

### 问题发现
- **现象**: v16.8实验中Agent完全静止（速度=0）
- **影响**: 所有Agent不移动，在可见期饿死

### 根因
`batched_agents.py`中`step()`方法的bug：
```python
# 原代码：当brain_fn=None时，输出全零！
if brain_fn is not None:
    brain_outputs = brain_fn(batch)
else:
    brain_outputs = torch.zeros(batch.n, n_outputs, device=self.device)
```

### 修复方案
```python
# 修复后：默认调用forward_brains
elif self.brain_matrix is not None and env is not None:
    sensors = self.get_sensors(env)
    brain_outputs = self.forward_brains(sensors)
```

### 验证结果
- 使用随机脑输出测试：Agent能够移动（位移~0.7/20步）
- 随机初始化的大脑矩阵输出接近零是正常的（稀疏随机图）
- 演化会逐渐产生有效的大脑连接

### 相关文件
- `core/eoe/batched_agents.py` - 已修复

## EOE v16.10-16.13 欺骗性景观实验

### 问题：伏地魔策略
Agent静止不动等待能量源接近，导致演化无法逼出复杂大脑

### 解决方案演进

#### v16.10：定居者税（失败）
- 策略：对静止>80步的Agent额外征税
- 结果：税收累积过快，种群崩溃

#### v16.11：认知溢价增强（部分成功）
- 策略：提高invisible奖励(5x) + 更高代谢(0.015)
- 结果：节点峰值7.6，复杂度82（历史新高！）
- 问题：税收仍导致后期崩溃

#### v16.12：降低税收（失败）
- 策略：tax_rate 0.01→0.003
- 结果：峰值复杂度82，但种群仍崩溃

#### v16.13：课程学习（成功！）✅
- 策略：移除定居者税 + 三阶段课程学习
  - 阶段I (0-1000步): 80%可见 - 伊甸园期
  - 阶段II (1000-3000步): 40%可见 - 筛选期
  - 阶段III (3000+): 15%可见 - 终极试炼

### 最终结果 (v16.13)
| Step | 种群 | 节点 | 复杂 |
|------|------|------|------|
| 200 | 63 | 6.7 | 52 |
| 600 | 86 | 6.3 | 67 |
| 1000 | 68 | 5.7 | 43 |
| 2000 | 47 | 6.11 | 45 |

**关键突破**：
- ✅ 移除定居者税成功 - 种群稳定在47
- ✅ 课程学习生效 - 复杂度后期回升到45
- ✅ 无人工惩罚 - 纯环境压力筛选

### 核心结论
1. 认知溢价机制完全有效（节点7.6，复杂度82峰值）
2. 人工税收有上限，环境压力无上限
3. 课程学习是引导复杂度的最佳方案
