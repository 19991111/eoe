# 2026-03-10 工作日志

## EOE项目 - 三大突破机制

### 专家建议的三个方案

1. **代谢疲劳 + 安全掩体** - 降维6步长链路
2. **无聊信息素** - 破解气味盲区与贴墙死循环
3. **夏日食物热力学** - 赋予"放下"即时收益

### 实施过程

- **问题1: 适应度悬崖**
  - 最初实现强制睡眠+20%死亡率
  - Agent在无法控制的情况下随机死亡，演化无法学习
  - 这是典型的"适应度悬崖"问题

- **解决方案: 渐进式物理压迫**
  1. 疲劳度影响速度（不强制睡眠）
  2. 静止3帧后自动恢复疲劳
  3. 睡觉消耗大量能量（起床饥饿）
  4. 睡眠时食物自动掉落（物理贮粮）

### 核心代码修改

- `environment.py`: 添加疲劳系统、信息素系统、食物热力学
- `agent.py`: 添加fatigue, max_fatigue, stationary_frames等属性
- `node.py`: 添加PHEROMONE_SENSOR

### Flag控制

```python
pop.environment.enable_fatigue_system(enabled=True, max_fatigue=50.0, fatigue_build_rate=0.5)
pop.environment.enable_pheromone_system(enabled=True)
pop.environment.enable_food_thermodynamics(enabled=True)
```

### 原则

- **只设计环境压力，不设计大脑结构**
- 智能必须从生存竞争中涌现
- 避免适应度悬崖，创造适应度缓坡

### 结果

- v0.97 最高适应度: 245.7
- 贮粮(Store): 1次出现
- 携带(Carry): 有出现
